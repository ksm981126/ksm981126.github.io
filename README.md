# 급여 달력

개인 근무 기록과 급여를 달력으로 관리하는 오프라인 우선 웹앱입니다. GitHub Pages에 올려서 PC와 모바일에서 같은 주소로 사용할 수 있고, Google Drive 파일 하나에 데이터를 저장해 기기 간 동기화할 수 있습니다.

## 주요 기능

- 달력에서 날짜를 눌러 출근 기록 입력
- 기본 시급, 출근/퇴근 시간, 휴게시간 설정
- 일별 시급, 출근 시간, 퇴근 시간, 휴게시간 수정
- 5인 미만/5인 이상 사업장 설정
- 주휴수당, 연장근로, 야간근로, 휴일근로 예상 계산
- 법정 근로시간 기준 주 단위 알림
- 주별 연장근로 시간 표시
- 공휴일, 대체공휴일, 주말 색상 표시
- 평일 공휴일 유급휴일 계산
- 월급일 설정 및 달력 표시
- 월급일이 공휴일/주말이면 이전 평일로 자동 조정
- 전월 근무분을 다음 달 월급일에 받는 방식으로 표시
- 4대보험, 소득세, 지방소득세 등 세후 예상
- 입사일 기준 연차/휴가 계산
- 휴가 사용 기록 및 남은 휴가 표시
- 월별 급여 조회
- 전체 조회에서 연간 급여와 월별 그래프 표시
- 업무일지 달력 전환
- 날짜별 현장명, 작업기간, 맡은 업무 기록
- 설치/점검/배터리 교체/UPS 교체/철거/유급휴가 업무 중복 선택
- 하루 여러 현장 기록
- 현장 업무가 없는 날의 사무실 기록
- 앱 잠금 비밀번호 설정
- JSON 파일 백업/가져오기
- Google Drive 저장/불러오기/자동 동기화
- 모바일 홈화면 추가 가능한 PWA 방식

## 사용 주소

현재 배포 주소:

```text
https://ksm981126.github.io/
```

다른 사람이 이 프로젝트를 복사해서 사용할 경우, 자기 GitHub Pages 주소로 바꿔서 사용하면 됩니다.

## 기본 사용 방법

1. 앱 주소에 접속합니다.
2. 처음 실행하면 왼쪽 설정 메뉴에서 기본 정보를 입력합니다.
   - 시급
   - 기본 출근 시간
   - 기본 퇴근 시간
   - 휴게시간
   - 월급일
   - 입사일
   - 사업장 규모
   - 세후 예상 사용 여부
3. 달력에서 근무한 날짜를 누릅니다.
4. 필요한 경우 그날의 시급, 출근/퇴근 시간, 휴게시간을 수정합니다.
5. 저장하면 달력에 출근 표시가 생기고 급여 예상에 반영됩니다.
6. `급여 조회`에서 월별 급여 또는 전체 조회 그래프를 확인합니다.
7. `업무일지`를 누르면 업무일지 달력으로 전환됩니다.
8. 업무일지 달력에서 날짜를 누르면 현장명, 작업기간, 맡은 업무를 기록할 수 있습니다.

## 모바일에서 사용하기

휴대폰 브라우저에서 앱 주소를 연 뒤 홈화면에 추가하면 앱처럼 사용할 수 있습니다.

### Android Chrome / Samsung Internet

1. 앱 주소 접속
2. 브라우저 메뉴 열기
3. `홈 화면에 추가` 선택
4. 홈화면 아이콘으로 실행

### iPhone Safari

1. 앱 주소 접속
2. 공유 버튼 선택
3. `홈 화면에 추가` 선택
4. 홈화면 아이콘으로 실행

## PC에서 사용하기

브라우저에서 앱 주소를 열어 사용하면 됩니다.  
Google Drive 동기화를 켜면 PC와 모바일이 같은 Drive 파일을 기준으로 데이터를 공유합니다.

## Google Drive 연동 방식

앱은 Google Drive에 아래 파일 하나를 만들고, 이 파일에 전체 데이터를 저장합니다.

```text
salary-calendar-data.json
```

저장되는 데이터:

- 출근 기록
- 휴가 기록
- 시급/근무시간/월급일/입사일 등 설정값
- 세후 계산 설정
- 잠금 비밀번호 정보

앱은 Google Drive 전체 파일을 읽는 방식이 아니라, 이 앱이 만든 파일에 접근하는 `drive.file` 권한을 사용합니다.

## Google Drive 연결 방법

1. 앱에서 `Google Drive 연결`을 누릅니다.
2. 본인 Google 계정으로 로그인합니다.
3. 권한을 허용합니다.
4. 처음 사용하는 경우 `Drive에 저장`을 눌러 현재 데이터를 저장합니다.
5. 다른 기기에서는 같은 Google 계정으로 `Google Drive 연결` 후 `Drive에서 불러오기`를 누릅니다.
6. 자동으로 저장하고 싶으면 `자동 동기화`를 켭니다.

## 다른 사람이 자기 계정으로 사용하려면

이 프로젝트를 그대로 복사해서 자기 GitHub Pages에 올릴 수 있습니다.  
다만 Google Drive 연동을 쓰려면 Google OAuth 설정을 자기 계정 기준으로 새로 만들어야 합니다.

### 1. GitHub Pages 준비

1. GitHub에서 새 저장소를 만듭니다.
2. 이 프로젝트 파일을 저장소에 올립니다.
3. GitHub Pages를 켭니다.
4. 배포 주소를 확인합니다.

예:

```text
https://username.github.io/
```

### 2. Google Cloud 프로젝트 만들기

1. Google Cloud Console에 접속합니다.
2. 새 프로젝트를 만듭니다.
3. Google Drive API를 사용 설정합니다.
4. OAuth 동의 화면을 설정합니다.
5. 테스트 단계라면 본인 Google 계정을 테스트 사용자로 추가합니다.

### 3. OAuth 클라이언트 ID 만들기

애플리케이션 유형은 아래로 선택합니다.

```text
웹 애플리케이션
```

승인된 JavaScript 원본에 자기 GitHub Pages 주소를 추가합니다.

예:

```text
https://username.github.io
```

주의:

- 끝에 `/`를 붙이지 않는 것을 권장합니다.
- `승인된 리디렉션 URI`는 이 앱에서는 보통 비워도 됩니다.
- 주소가 다르면 Google 로그인이 막힐 수 있습니다.

### 4. 앱의 클라이언트 ID 바꾸기

`app.js`에서 아래 값을 자기 OAuth 클라이언트 ID로 바꿉니다.

```js
const GOOGLE_CLIENT_ID = "여기에_본인_클라이언트_ID";
```

그 뒤 다시 GitHub Pages에 올리면 자기 Google Drive에 연결해서 사용할 수 있습니다.

## 데이터 백업과 이동

### Google Drive로 이동

새 휴대폰이나 새 PC에서:

1. 앱 주소 접속
2. Google Drive 연결
3. `Drive에서 불러오기`

### JSON 백업 파일로 이동

기존 기기에서:

1. `백업 내보내기`
2. JSON 파일 저장

새 기기에서:

1. `가져오기`
2. 백업 JSON 파일 선택

## 오프라인과 캐시

이 앱은 오프라인에서도 열 수 있도록 서비스 워커 캐시를 사용합니다.  
새 버전이 배포되면 이전 오프라인 캐시는 자동으로 삭제되도록 구성되어 있습니다.

화면이 예전 버전으로 계속 보이면:

1. 앱을 완전히 종료 후 다시 열기
2. 그래도 안 되면 홈화면 아이콘 삭제 후 다시 추가
3. 마지막으로 브라우저의 사이트 데이터에서 `ksm981126.github.io` 데이터를 삭제

주의: 사이트 데이터를 삭제하면 그 기기에만 저장된 로컬 데이터는 사라질 수 있습니다. Google Drive에 저장한 데이터는 유지됩니다.
